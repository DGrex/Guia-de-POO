El Constructor __init__

Aprende a usar el método especial __init__ para inicializar objetos y entender cómo Python gestiona los atributos de instancia.

Progreso
1 / 20

¿Qué es el constructor __init__?

En Python, __init__ es un método especial (también llamado "dunder method") que se ejecuta automáticamente cada vez que se crea una instancia de una clase. Su propósito principal es inicializar el estado interno del objeto.

📌
Nota clave El método __init__ no crea el objeto (eso lo hace __new__), sino que lo inicializa asignando valores iniciales a sus atributos.
python 
# Definimos una clase Persona con el constructor __init__
class Persona:
    def __init__(self, nombre: str, edad: int):
        # 'self' hace referencia al objeto que se está creando
        self.nombre = nombre   # atributo de instancia
        self.edad = edad       # atributo de instancia

    def presentarse(self) -> str:
        return f"Hola, soy {self.nombre} y tengo {self.edad} años."


# Al crear objetos, Python llama __init__ automáticamente
p1 = Persona("Ana", 25)
p2 = Persona("Luis", 30)

print(p1.nombre)          # Salida: Ana
print(p2.edad)            # Salida: 30
print(p1.presentarse())   # Salida: Hola, soy Ana y tengo 25 años.
💡
¿Por qué usar self? self es una referencia al objeto actual. Aunque su nombre es convención (puedes usar cualquier nombre), es altamente recomendable seguir el estándar.

Valores por defecto en __init__

Puedes definir valores por defecto para los parámetros del constructor, lo que hace los argumentos opcionales al crear instancias:

python 
class Producto:
    def __init__(self, nombre: str, precio: float = 0.0, stock: int = 0):
        self.nombre = nombre
        self.precio = precio
        self.stock  = stock

    def __repr__(self) -> str:
        return f"Producto({self.nombre!r}, precio={self.precio}, stock={self.stock})"


# Con todos los argumentos
cafe = Producto("Café", 2.50, 100)

# Solo con el nombre (usa valores por defecto)
nuevo = Producto("Sin definir")

print(cafe)    # Producto('Café', precio=2.5, stock=100)
print(nuevo)   # Producto('Sin definir', precio=0.0, stock=0)

Practica los conceptos aprendidos con estos ejercicios comentados paso a paso.

Ejercicio 01 Crea una clase Coche con atributos marca y modelo
python 
class Coche:
    def __init__(self, marca: str, modelo: str):
        # Guardamos marca y modelo como atributos de instancia
        self.marca  = marca
        self.modelo = modelo

    def descripcion(self) -> str:
        return f"{self.marca} {self.modelo}"


# Creamos dos coches con distintos valores
c1 = Coche("Toyota", "Corolla")
c2 = Coche("Honda", "Civic")

print(c1.marca)          # Toyota
print(c2.modelo)         # Civic
print(c1.descripcion())  # Toyota Corolla
Ejercicio 02 ¿Qué pasa si no defines __init__?
⚠️
Si no defines __init__, Python usa el constructor heredado de object que no inicializa ningún atributo personalizado.
python 
class ClaseSinInit:
    pass   # Sin __init__


obj = ClaseSinInit()  # Funciona, pero sin atributos

# Podemos asignar atributos manualmente (mala práctica)
obj.nombre = "Ejemplo"
print(obj.nombre)  # Ejemplo

# Sin embargo, esto es inconsistente y difícil de mantener.
# ✅ Siempre es mejor definir __init__ para garantizar
# que todos los objetos tengan los mismos atributos.

Conceptos Clave

📐
Resumen visual del constructor
  • __init__(self, ...) se ejecuta al crear un objeto con Clase(args)
  • self representa la instancia del objeto actual
  • Los atributos definidos con self.attr son atributos de instancia
  • Puedes definir valores por defecto para parámetros opcionales
