Abstracción Encapsulamiento Herencia Polimorfismo
💡
El paradigma POO: Modelar software basándose en clases y objetos permite agrupar datos (atributos) y comportamientos (métodos) en entidades únicas y autocontenidas, facilitando la escalabilidad, modularidad y el mantenimiento a largo plazo de aplicaciones complejas.
Teoría

Los 4 Pilares y Conceptos Avanzados

1. Abstracción

Concepto: Consiste en aislar y mostrar únicamente los elementos esenciales al usuario, ocultando los complejos procesos lógicos o mecánicos internos detrás de una interfaz simple y clara.

abstraccion.py
class Calculadora:
    def sumar(self, a, b):
        return a + b

calc = Calculadora()
print(calc.sumar(5, 3))  # El usuario solo invoca el método. Salida: 8

2. Encapsulamiento

Concepto: Restringe el acceso directo a los datos y componentes internos de un objeto para prevenir modificaciones accidentales o maliciosas, forzando a que las alteraciones pasen por métodos de control perimetral.

encapsulamiento.py
class Cuenta:
    def __init__(self):
        self._saldo = 0  # Guion bajo (_) denota convención de atributo protegido

    def depositar(self, valor):
        self._saldo += valor

    def ver_saldo(self):
        return self._saldo

c = Cuenta()
c.depositar(100)
print(c.ver_saldo())  # Salida: 100

3. Herencia

Concepto: Mecanismo jerárquico que permite a una clase secundaria o "hija" adquirir e implementar los atributos y rutinas de una superclase o "padre", promoviendo la reutilización de código.

herencia.py
class Persona:
    def __init__(self, nombre):
        self.nombre = nombre

class Cliente(Persona):
    def __init__(self, nombre, correo):
        super().__init__(nombre)  # Inicializa la superclase
        self.correo = correo

cli = Cliente("Daniel", "d@mail.com")
print(cli.nombre, cli.correo)  # Salida: Daniel d@mail.com

4. Polimorfismo

Concepto: Capacidad que poseen diferentes objetos de responder al mismo llamado o firma de método implementando comportamientos internos morfológicamente distintos según su naturaleza.

polimorfismo.py
class Animal:
    def sonido(self): pass

class Perro(Animal):
    def sonido(self): print("Ladra")

class Gato(Animal):
    def sonido(self): print("Maúlla")

class Vaca(Animal):
    def sonido(self): print("Muge")

for animal in [Perro(), Gato(), Vaca()]:
    animal.sonido()

5. Manejo de Propiedades Dinámicas (@property)

Concepto: Permite transformar métodos en atributos legibles directos, interceptando sus lecturas, mutaciones (setters) y eliminaciones para inyectar validaciones o formateos en tiempo real.

propiedades.py
class Producto:
    def __init__(self, nombre, precio):
        self._nombre = nombre
        self._precio = precio

    @property
    def precio(self):
        return self._precio

    @precio.setter
    def precio(self, valor):
        if valor < 0:
            raise ValueError("Precio inválido: no puede ser negativo")
        self._precio = valor

p = Producto("Laptop", 900)
p.precio = 1200  # Invoca automáticamente al @precio.setter
print(p.precio)  # Invoca automáticamente al @property getter. Salida: 1200

6. Clases Abstractas e Interfaces de Diseño (ABC)

Concepto: Clases estructurales de guía que no admiten instancias directas. Actúan como contratos estrictos de arquitectura, obligando a las clases herederas a codificar de forma explícita los métodos decorados con @abstractmethod.

clases_abstractas.py
from abc import ABC, abstractmethod

class Figura(ABC):
    @abstractmethod
    def area(self): pass

class Triangulo(Figura):
    def __init__(self, base, altura):
        self.base = base
        self.altura = altura
        
    def area(self):  # Implementación obligatoria de la firma del contrato
        return (self.base * self.altura) / 2

t = Triangulo(10, 5)
print(t.area())  # Salida: 25.0

Ejercicio 1: Cumplimiento de Abstracción Estricta contractual

Crea una jerarquía de diseño geométrico obligando a la subclase a resolver ecuaciones matemáticas perimetrales complejas.

figura_triangulo.py 
from abc import ABC, abstractmethod

class Figura(ABC):
    @abstractmethod
    def area(self): pass

class Triangulo(Figura):
    def __init__(self, base, altura):
        self.base = base
        self.altura = altura
    def area(self):
        return (self.base * self.altura) / 2

t = Triangulo(10, 5)
print(t.area())  # Salida: 25.0
