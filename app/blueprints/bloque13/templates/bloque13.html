decorador_basico.py 
def mi_decorador(funcion_original):
    def wrapper(*args, **kwargs):
        print("Antes de ejecutar")
        resultado = funcion_original(*args, **kwargs)
        print("Después de ejecutar")
        return resultado
    return wrapper

@mi_decorador
def saludar():
    print("Hola mundo")

saludar()

💡
Funciones de Orden Superior: En Python, las funciones son objetos de "primera clase", lo que significa que pueden ser pasadas como argumentos, asignadas a variables y retornadas por otras funciones. Un decorador saca provecho de esto envolviendo una función (callback) dentro de una función interna (wrapper) para alterar sus acciones sin modificar su código fuente original.

Caso de Uso Práctico: Inyección de Validaciones Perimetrales

Uno de los mayores beneficios de los decoradores es la separación de conceptos. Podemos aislar las reglas lógicas de control sanitario (como evitar divisiones matemáticas indeterminadas) de la implementación de la función core aritmética:

decorador_validacion.py 
def validar_division(funcion):
    def wrapper(a, b):
        if b == 0:
            print("Error: división por cero")
            return None
        return funcion(a, b)
    return wrapper

@validar_division
def dividir(a, b):
    return a / b

print(dividir(10, 2))  # Salida: 5.0
print(dividir(10, 0))  # Salida: Error: división por cero -> None

Ejercicio 1: Intercepción simple de flujos

Crea un decorador básico que actúe como un disparador de inicialización imprimiendo una alerta previa en terminal antes del llamado.

decorador_simple.py 
def iniciar(funcion):
    def wrapper(*args, **kwargs):
        print("Iniciando ...")
        return funcion(*args, **kwargs)
    return wrapper

@iniciar
def hola():
    print("Hola Denis")

hola()
# Salida:
# Iniciando ...
# Hola Denis

Ejercicio 2: Validación condicional de parámetros de entrada

Desarrolla una envoltura protectora que impida el procesamiento algebraico si los argumentos de entrada no cumplen con criterios estrictamente positivos.

decorador_filtro.py 
def positivo(funcion):
    def wrapper(x):
        if x <= 0:
            print("Error: debe ser positivo")
            return None
        return funcion(x)
    return wrapper

@positivo
def cuadrado(x):
    return x ** 2

print(cuadrado(4))   # Salida: 16
print(cuadrado(-3))  # Salida: Error: debe ser positivo

Ejercicio 3: Sistema básico de trazas (Logger)

Diseña una rutina de monitoreo automático que registre el llamado de funciones operacionales inyectando mensajes informativos en tiempo de ejecución.

decorador_log.py 
def log(funcion):
    def wrapper(*args, **kwargs):
        print("Llamando función ...")
        return funcion(*args, **kwargs)
    return wrapper

@log
def suma(a, b):
    return a + b

print(suma(2, 3))
# Salida:
# Llamando función ...
# 5
