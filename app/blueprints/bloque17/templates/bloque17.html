patron_mixin.py 
class ValidacionMixin:
    """Clase auxiliar - No se instancia por sí sola."""
    def validar_nombre(self, nombre):
        if not nombre: raise ValueError("Nombre vacío")

    def validar_edad(self, edad):
        if edad < 0: raise ValueError("Edad inválida")

class SistemaEstudiantes(ValidacionMixin):
    def __init__(self):
        self.estudiantes = []

    def registrar(self, nombre, edad):
        self.validar_nombre(nombre)  # Inyectado desde el Mixin
        self.validar_edad(edad)      # Inyectado desde el Mixin
        self.estudiantes.append({"nombre": nombre, "edad": edad})
class Mixin __mro__ Herencia Múltiple C3 Linearization
💡
¿Qué es un Mixin? Es un patrón de diseño orientado a objetos que permite "inyectar" funcionalidades específicas en clases derivadas mediante herencia múltiple. Un Mixin no está pensado para instanciarse de forma aislada ni para mantener un estado interno complejo; su único propósito es agrupar métodos utilitarios modulares y evitar la duplicación de código (DRY).

MRO: Orden de Resolución de Métodos

Cuando trabajas con herencia múltiple en Python, múltiples clases padre pueden definir métodos con el mismo nombre. Python resuelve esta ambigüedad de manera predecible utilizando el algoritmo **C3 Linearization** para construir el MRO (Method Resolution Order), recorriendo las superclases estrictamente de izquierda a derecha de acuerdo a su declaración en la firma de la subclase:

mro_lookup.py 
class A:
    def metodo(self): 
        print("Ejecutando desde: A")

class B:
    def metodo(self): 
        print("Ejecutando desde: B")

# Al listar primero a 'A', tendrá prioridad en la cadena de búsqueda
class C(A, B):
    pass

C().metodo()  # Salida: Ejecutando desde: A
print(C.__mro__) # Muestra el orden jerárquico completo de resolución

Ejercicio 1: Sanitización e inspección de documentos de identidad

Implementa una clase de validación auxiliar encargada de verificar la integridad estructural de identificaciones numéricas antes del guardado en el sistema.

mixin_validacion.py 
class ValidacionMixin:
    def validar_cedula(self, cedula):
        if len(cedula) != 10 or not cedula.isdigit():
            raise ValueError("Cédula inválida")

class SistemaEstudiantes(ValidacionMixin):
    def registrar(self, nombre, edad, cedula):
        self.validar_cedula(cedula)  # Consumo del método inyectado
        print(f"Estudiante {nombre} registrado con cédula {cedula}")

sistema = SistemaEstudiantes()
sistema.registrar("Ana", 22, "1234567890")
# Salida: Estudiante Ana registrado con cédula 1234567890

Ejercicio 2: Inyección de capacidades de agregación estadística

Extiende el alcance operativo de una clase de datos inyectándole un algoritmo reutilizable para el cálculo de promedios aritméticos.

mixin_computo.py 
class PromedioMixin:
    def calcular_promedio(self, notas):
        return sum(notas) / len(notas)

class Estudiante(PromedioMixin):
    def __init__(self, nombre, notas):
        self.nombre = nombre
        self.notas = notas

e = Estudiante("Luis", [10, 8, 9])
print(e.calcular_promedio(e.notas))  # Salida: 9.0

Ejercicio 3: Comportamiento jerárquico frente a colisión de nombres

Analiza cómo el intérprete de Python descarta o prioriza métodos homónimos basándose en el orden secuencial estricto de la tupla de herencia de la firma.

Concepto Clave de Resolución: Si una clase hereda de forma múltiple mediante la sintaxis class SubClase(ClaseA, ClaseB) y ambas superclases implementan un método idéntico, Python interceptará y despachará la rutina correspondiente a la primera clase listada (en este caso, ClaseA). Este orden lineal evita colisiones imprevistas y dota de predictibilidad al sistema de herencia múltiple.
