Inyecta capacidades dinámicas de auto-exploración en tus objetos en tiempo de ejecución. Comprende cómo auditar y listar estructuras de clases dominando el uso técnico de métodos mágicos y funciones del sistema como vars(), hasattr() y getattr() mediante herencia auxiliar.

mostrar_mixin.py 
class MostrarMixin:
    def mostrar(self, atributo=None):
        print(f"Clase: {self.__class__.__name__}")
        if atributo:
            if hasattr(self, atributo):
                print(f"{atributo}: {getattr(self, atributo)}")
            else:
                print(f"Atributo '{atributo}' no existe en {self.__class__.__name__}")
        else:
            for nombre, valor in vars(self).items():
                print(f"{nombre}: {valor}")

💡
Concepto de Mixin: Un Mixin es una clase auxiliar diseñada exclusivamente para proveer un comportamiento empaquetado y reutilizable a otras subclases mediante herencia. No busca modelar una relación de identidad ("es un"), sino añadir capacidades opcionales y transversales al ecosistema de tus objetos de software.
Teoría

Herramientas de Auditoría Interna

1. Identificación Dinámica de Clase

Concepto: Mediante el acceso a los metadatos de la instancia a través del atributo mágico de sistema self.__class__.__name__, el programa puede conocer el string exacto del molde original desde el cual se instanció el objeto en ejecución.

meta_clase.py
class Ejemplo:
    pass

obj = Ejemplo()
print(obj.__class__.__name__)  # Salida: Ejemplo

2. Mapeo del Estado Completo (Función vars)

Concepto: La función integrada vars(objeto) expone el diccionario interno o mapa de variables (__dict__) de la instancia. Esto permite iterar de forma dinámica sobre las llaves (nombres de atributos) y valores actuales guardados en memoria.

mapeo_vars.py
class Producto:
    def __init__(self, sku, precio):
        self.sku = sku
        self.precio = precio

p = Producto("A100", 19.99)
print(vars(p))  # Salida: {'sku': 'A100', 'precio': 19.99}

3. Extracción Segura mediante Cadenas

Concepto: Las funciones hasattr() y getattr() permiten comprobar la presencia de una propiedad y extraer su valor usando un parámetro string. Esto ayuda a crear consultas lógicas flexibles sin arrojar excepciones críticas de tipo AttributeError.

extraccion_segura.py
# Uso del MostrarMixin heredado por un Cliente corporativo
class Cliente(MostrarMixin):
    def __init__(self, nombre, correo):
        self.nombre = nombre
        self.correo = correo

c = Cliente("Luis", "luis@mail.com")
c.mostrar("correo")  # Salida aislando el atributo: correo: luis@mail.com

Ejercicio 1: Herencia estructural de Mixin

Crea una clase Persona que herede de MostrarMixin y tenga atributos nombre y edad. Usa mostrar() para imprimirlos en consola.

clase_persona.py 
class Persona(MostrarMixin):
    def __init__(self, nombre, edad):
        self.nombre = nombre
        self.edad = edad

p1 = Persona("Carlos", 30)
p1.mostrar()
# Salida:
# Clase: Persona
# nombre: Carlos
# edad: 30
